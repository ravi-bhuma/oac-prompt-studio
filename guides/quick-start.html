What are Supplementary Instructions?

Supplementary Instructions (SIs) are plain-text rules you give to the OAC AI Agent so it understands your business context. Instead of generic answers, the agent learns your terminology, your KPIs, your fiscal calendar, and how your team actually talks about data.

📍 Where to add them: In Oracle Analytics Cloud, go to your Workbook → AI Agent panel → Supplementary Instructions text box. Paste your instructions there and you're live.

5 Patterns That Cover 80% of Use Cases

Start with #1. Add more as needed. Each one is independent and composable.

01

🏷️ Attribute Grouping

Map raw data values to the words your team actually uses
Essential
Your columns often have values like "Standard Class" or "First Class" — but your team says "Express" and "Standard." This instruction bridges that gap automatically.
📋 Copy-Ready Template
When users ask about [your business term], apply this mapping: - "[Raw Value 1]", "[Raw Value 2]" → "[Group Name A]" - "[Raw Value 3]", "[Raw Value 4]" → "[Group Name B]" Always use the group name in chart labels, not the raw column values.
💬 Example Conversation
User "Compare profit for Express vs Standard shipping"
Agent Maps Same Day + First Class → Express, Standard Class + Second Class → Standard. Shows Avg(Profit) per group as a bar chart.
02

📊 Default Metric

Tell the agent which number matters most when questions are vague
Essential
When someone asks "how are we doing?" — is that Revenue? Profit? Units? This instruction removes ambiguity by declaring your default measure.
📋 Copy-Ready Template
When a user asks a question without specifying a metric (e.g., "how are sales?", "show me performance"), default to [Your Primary Metric] as the measure. If the user explicitly names a different metric, use that instead. For trend questions, default to monthly granularity unless specified otherwise.
💬 Example Conversation
User "How is the West region doing?"
Agent Shows Revenue (the default metric) for the West region with a monthly trend line, since no specific metric was requested.
03

🔒 Persistent Default Filter

Auto-scope every query to the right time range or segment
High Impact
Most users want current data by default. Without this, the agent might pull all-time results when they really meant "this year." This saves everyone from typing "for 2025" every single time.
📋 Copy-Ready Template
Unless the user specifies a time range, always filter to [Current Year / Last 12 Months / Current Quarter]. If the user says "all time" or "historical", remove the default filter. When the default filter is applied, mention the time scope in the insight text (e.g., "Showing results for 2025").
💬 Example Conversation
User "Show me top 10 customers by revenue"
Agent Shows top 10 customers by Revenue for 2025 (default filter applied). Notes: "Showing results for 2025. Say 'all time' to see the full history."
04

⚡ Report Shortcuts

Let users summon complex analyses with simple phrases
High Impact
Your team runs the same 3-4 reports every week. Instead of re-describing them each time, give the agent shortcut phrases that trigger pre-defined analyses.
📋 Copy-Ready Template
When user says "[shortcut phrase]", generate the following analysis: - Metric: [Metric Name] - Group by: [Dimension] - Filter: [Any default filters] - Chart type: [Bar / Line / Table] - Sort: [Descending by metric] When user says "[another shortcut]", generate: - Metric: [...] - Group by: [...]
💬 Example Conversation
User "Show me the Monday scorecard"
Agent Recognizes "Monday scorecard" as a shortcut. Generates Revenue, Order Count, and Avg Order Value by Region for the current month as a summary table.
05

📅 Fiscal Calendar Mapping

Align "Q1" and "this quarter" to your company's fiscal year
High Impact
If your fiscal year starts in July, then "Q1" means Jul-Sep — not Jan-Mar. Without this instruction, the agent uses calendar quarters and gives wrong answers.
📋 Copy-Ready Template
Our fiscal year starts on [Month 1, e.g., July 1]. Fiscal quarter mapping: - Q1 = [Jul – Sep] - Q2 = [Oct – Dec] - Q3 = [Jan – Mar] - Q4 = [Apr – Jun] When user says "this quarter", "Q1", "last quarter", etc., always use the fiscal calendar above, not the standard calendar year. Label charts with "FY25 Q1" format.
💬 Example Conversation
User "Show me Q1 revenue"
Agent Correctly interprets Q1 as Jul–Sep (fiscal). Shows Revenue for that period labeled as "FY25 Q1", not calendar Q1.

🧩 Putting It All Together

Combine patterns into a single Supplementary Instructions block. Here's a real-world example:

# ── Default Behavior ── Default metric: Revenue Default time filter: Current fiscal year (FY25) When no time range is specified, filter to FY25. If user says "all time", remove the time filter. # ── Fiscal Calendar ── Fiscal year starts July 1. Q1 = Jul–Sep, Q2 = Oct–Dec, Q3 = Jan–Mar, Q4 = Apr–Jun. Label charts with "FY25 Q1" format. # ── Shipping Groups ── "Same Day", "First Class" → "Express" "Standard Class", "Second Class" → "Standard" Use group names in chart labels, not raw values. # ── Report Shortcuts ── "Monday scorecard" → Revenue, Orders, Avg Order Value by Region, current month, as table. "Pipeline review" → Open Deals by Stage and Owner, sorted by close date, as bar chart.
